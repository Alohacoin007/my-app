## 1. 페이지 해부도 — robinhood.com/us/en/about/crypto/

### 1.0 페이지의 설득 문법 (전체 구조 요약)
스니펫으로 확정된 콘텐츠 블록 + 로빈후드 마케팅 페이지 관례로 재구성한 섹션 순서:

```
[헤더: 블랙 바] → [히어로: 초대형 헤드라인 + 단일 CTA] → [증거 스탯: "3.6% more crypto"]
→ [기능 밴드 1: 24/7 트레이딩 + 코인 카탈로그] → [기능 밴드 2: 스테이킹]
→ [기능 밴드 3: 전송(입출금 수수료 0)] → [기능 밴드 4: 트레이딩 도구]
→ [신뢰 섹션: 보안(콜드 스토리지·범죄보험·외부 감사)] → [최종 CTA] → [각주 디스클로저 블록] → [푸터]
```
핵심 문법 = **"주장 → 제3자 검증 각주"의 반복**. 모든 수치 주장(최저비용·3.6%)에 위첨자 각주가 붙고 페이지 하단에 검증 조건("independently verified by a third party, using quotes for BTC/ETH/DOGE as of 5/18/2026")이 산다. 규제 금융 마케팅의 전형.

### 1.2 히어로 [부분 확정]

## 3. ALPEXA 번역안

### 3.0 전제 (현행 실측)
- **fx.html** (911줄): 헤더 v2(로빈후드 컨셉) 적용 완료 · 히어로 = 초대형 숫자 슬라이드 3장("0.0 Tight spreads / 500:1 Max leverage / 80 Liquidity banks") + 도트 + 듀얼 CTA · 티커 테이프 · LP 뱅크 스트립 · Markets 3카드 · Account tiers 3단 · Trust 4열 · lp-cta · 워드마크 푸터. 라이트 강제(`body.light`), 액센트 #1e40d8, 버튼 radius 0(각형), 디스플레이 폰트 Space Grotesk.
- **markets.html** (404줄): 구형 화이트 헤더(블랙 바 아님) · Google Finance식 워치리스트+차트 터미널.
- **about.html** (224줄): 구형 화이트 헤더 · CEO 레터 · 가치 4카드 · 팩트 스트립 · 최종 CTA.

### 3.1 번역 원칙
1. **이미 정합인 것 (변경 불필요):** 초대형 숫자 히어로(fx.html의 big0 = 로빈후드 스탯 문법과 동일 계열), 각주식 리스크 바(resp-bar), Trust 4열, 흑 헤더. ALPEXA 액센트 #1e40d8 유지 — Robin Neon 이식 금지(상표 무드 차용 경계 + 우리 브랜드).
2. **가져올 것 = 구조·규율:** ① 주장→각주 규율, ② 헤더 일관성(전 페이지 동일 블랙 바), ③ 기능 zebra 밴드, ④ 신뢰 섹션의 구체성(콜드 스토리지급 "구체적 사실" 화법), ⑤ 여백 확대.
3. **가져오지 말 것:** 세리프 헤드라인 전면 도입(브랜드 대수술 — Space Grotesk 유지가 안전), Robin Neon 색, 정물 사진(제작 리소스 없음 — 제품 UI 크롭으로 대체).

## 4. 불확실/차단 목록 + 캡처 요청

### [차단] — 확정 실패 항목
1. 히어로 실제 H1 문구·레이아웃(텍스트 좌/중앙, 제품 렌더 유무) — 타이틀 태그만 확정.
2. 히어로 배경이 다크인지 오프화이트인지.
