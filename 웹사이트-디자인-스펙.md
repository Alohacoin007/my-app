## 1. 페이지 해부도 — robinhood.com/us/en/about/crypto/

### 1.0 페이지의 설득 문법 (전체 구조 요약)
스니펫으로 확정된 콘텐츠 블록 + 로빈후드 마케팅 페이지 관례로 재구성한 섹션 순서:

```
[헤더: 블랙 바] → [히어로: 초대형 헤드라인 + 단일 CTA] → [증거 스탯: "3.6% more crypto"]
→ [기능 밴드 1: 24/7 트레이딩 + 코인 카탈로그] → [기능 밴드 2: 스테이킹]
→ [기능 밴드 3: 전송(입출금 수수료 0)] → [기능 밴드 4: 트레이딩 도구]
→ [신뢰 섹션: 보안(콜드 스토리지·범죄보험·외부 감사)] → [최종 CTA] → [각주 디스클로저 블록] → [푸터]
```
핵심 문법 = **"주장 → 제3자 검증 각주"의 반복**. 모든 수치 주장(최저비용·3.6%)에 위첨자 각주가 붙고 페이지 하단에 검증 조건("independently verified by a third party, using quotes for BTC/ETH/DOGE as of 5/18/2026")이 산다. 규제 금융 마케팅의 전형.

## 3. ALPEXA 번역안

### 3.0 전제 (현행 실측)
- **fx.html** (911줄): 헤더 v2(로빈후드 컨셉) 적용 완료 · 히어로 = 초대형 숫자 슬라이드 3장("0.0 Tight spreads / 500:1 Max leverage / 80 Liquidity banks") + 도트 + 듀얼 CTA · 티커 테이프 · LP 뱅크 스트립 · Markets 3카드 · Account tiers 3단 · Trust 4열 · lp-cta · 워드마크 푸터. 라이트 강제(`body.light`), 액센트 #1e40d8, 버튼 radius 0(각형), 디스플레이 폰트 Space Grotesk.
- **markets.html** (404줄): 구형 화이트 헤더(블랙 바 아님) · Google Finance식 워치리스트+차트 터미널.
- **about.html** (224줄): 구형 화이트 헤더 · CEO 레터 · 가치 4카드 · 팩트 스트립 · 최종 CTA.

### 3.1 번역 원칙
1. **이미 정합인 것 (변경 불필요):** 초대형 숫자 히어로(fx.html의 big0 = 로빈후드 스탯 문법과 동일 계열), 각주식 리스크 바(resp-bar), Trust 4열, 흑 헤더. ALPEXA 액센트 #1e40d8 유지 — Robin Neon 이식 금지(상표 무드 차용 경계 + 우리 브랜드).
2. **가져올 것 = 구조·규율:** ① 주장→각주 규율, ② 헤더 일관성(전 페이지 동일 블랙 바), ③ 기능 zebra 밴드, ④ 신뢰 섹션의 구체성(콜드 스토리지급 "구체적 사실" 화법), ⑤ 여백 확대.
3. **가져오지 말 것:** 세리프 헤드라인 전면 도입(브랜드 대수술 — Space Grotesk 유지가 안전), Robin Neon 색, 정물 사진(제작 리소스 없음 — 제품 UI 크롭으로 대체).

## 4. 불확실/차단 목록 + 캡처 요청

### [차단] — 확정 실패 항목
1. 히어로 실제 H1 문구·레이아웃(텍스트 좌/중앙, 제품 렌더 유무) — 타이틀 태그만 확정.
2. 히어로 배경이 다크인지 오프화이트인지.

### 5.2 다크 이음새 전략

**위 (히어로 → 스트립):**
- 이음새의 주인공은 **행성 지평선 자체**. `overflow:hidden`이 글로우를 히어로 안에 가두므로(box-shadow도 -60px 위쪽으로만), 스트립은 아무 방어 장치 없이 `#000`이면 된다 — 밝은 지평선 → 순흑 낙차가 곧 구분.
- 스트립에 top 보더·그림자·그라디언트를 **추가하지 말 것** — 지평선 광원과 경쟁하는 모든 요소는 노이즈.
- 초광폭 뷰포트(>1640px) 주의: 원 지름 1640px 고정 → 지평선이 화면 양끝까지 안 닿고 가장자리는 #0B0C09 평면으로 끝남. 이 경우에도 #0B0C09→#000 이음은 지각 불가라 문제 없음. (원 크기를 vw 연동으로 키우는 것은 히어로 소관 — 본 스펙 범위 외, 메모만.)

**아래 (스트립 → darkband):**
- 같은 #000이므로 이음새 자체가 소멸 — `.lpticker`가 사실상 darkband의 서곡이 된다. features의 `padding-top:96px`가 스트립과 첫 섹션 헤드 사이 호흡을 담당.
- 결과 흐름: `블랙 헤더 → 다크 티커 → 블랙 히어로(지평선 광원) → [연속 블랙: LP 스트립 + features + vip + trust] → 페리윙클 CTA → 푸터`. 페이지 전체에서 밝은 면은 지평선·페리윙클 CTA 두 곳으로 압축 — 레전드 "근검은 + 절제된 광원" 무드.

### 6.4 구현 계획 (메인 세션용 — 백엔드 0변경)

**1단계 — 다크 전환 + 타이틀 행 + 탭 레일 (~140줄, markets.html 단일 파일):**
1. CSS 변수 블록 치환 (§6.3, 12변수) + footer/html 다크 + focus-visible 색.
2. JS `COL` 상수 1오브젝트 신설 → 하드코딩 7곳 치환 (§6.1 인벤토리 — **이걸 빼먹으면 "다크 페이지에 라이트 차트" 반쪽 전환**, 최대 위험).
3. 타이틀 행 마크업 신설(죽은 `.mkt-head` CSS 부활·수정) + `.mkt-tabs`를 쉘 위 레일로 이동(switchTab 재사용) + Metals 필터 탭(`tab=metals`, 기존 3값 무변).
4. `.m-tab.on`·`.m-tf.on`·`.m-cta` 네온 반전 (배경 #CDF138일 때 텍스트 반드시 #101418 — 흰 텍스트 잔존 시 대비 붕괴).

**2단계 — 히어로 캐러셀 (~170줄):**
5. `FEATURED` 배열 + `renderHero()`(킥커/이름/가격/차트/필/컨트롤) + `heroSpark()`(sparkSVG 확대 변형: 라인 2px·도트 r3·그리드 0) + 6s 타이머(호버/⏸/reduced-motion 정지).
6. `pull()` 말미 `updateHero()` 훅 1줄 — in-place 패치 (updateSide 패턴).
7. (선택) 틱 플래시 클래스 — 히어로 가격·행 가격 공용.

**위험 목록:**
- JS 색 하드코딩 7곳 (§6.1 표) — 1단계 2번이 전제조건.
- `?tab=` 딥링크 3값 파손 금지 (fx.html mcard 3건 실사용).
- TD 무료 키 한도 — 히어로에서 `loadChart()` 호출 절대 금지 (hist만 사용).
- 필에 bid/ask 표기 금지 — spr_pts 단위 해석 중복 구현 = 락스텝 위반 소지 (CLAUDE.md 스프레드 규칙).
- Indices: CAT 0심볼 — 탭·히어로에 노출 금지 (가짜 상품 노출 = 오즈 불변식과 동일 클래스 결함).

**게이트:** 각 단계 후 `node tests/verify.js` (UI 변경 필수). 스포츠 앱 파일 아님 → feed-check 비해당. 실브라우저에서 ① 3초 틱 갱신 ② 탭 4종+딥링크 3종 ③ 캐러셀 회전·정지 ④ TD 차트 5TF 확인 후 "완료".

### 6.5 불확실 항목
1. 캐러셀 자동 순환 주기 — 샘플 스크린샷은 정지 화면이라 주기 미상 → 6s [ESTIMATE], 사용자 체감 조정.
2. 히어로 필의 solid vs tint — 샘플은 solid, 본 스펙은 tint 권고(절제) → 시안 2종 렌더 후 사용자 택1 권장.
3. `webtrade.html` 딥링크 쿼리(심볼·사이드) 지원 여부 미확인 — 미지원이면 1차는 무파라미터 링크.
4. 헤더 드롭다운 "Indices" 카피(147) vs CAT 무심볼 불일치 — ① 카피에서 제거 or ② `prices`에 지수 심볼 실존 확인 후 CAT 추가(TD 심볼 매핑 필요). 사용자 결정.
5. FX 가격 pip 첨자 표기 — 샘플 % 첨자의 번역인데 FX 관례(마지막 자리 첨자 vs 가운데 2자리 확대)가 갈림 → 실렌더 후 확정.
6. `--m-panel2` #151A21 · 그라디언트 α · 간격값 — 전부 실렌더 스크린샷 1장으로 확정 승격 가능.
