# 북오피스 (Book Office) — 스포츠북의 두뇌

> **북오피스 = 스포츠북 사령탑** (노출·리스크·손익·정산감사). 고객/승인/정산을 *처리*하는
> **백오피스(Back office, `manager-mobile.html`)** 와 다르다. 북오피스는 **"북이 안 깨졌나(감사) +
> 돈 버나(hold%) + 어디 물렸나(노출)"** 를 본다.

## 🔴 P0 — 없으면 "북을 운영하는 게 아님"

### 1. 실시간 노출 뷰 (Liability) ← **최우선, 데이터 이미 있음**
- **무엇:** 경기(gid)×결과별 **순지급액** = "Lakers 이기면 우리가 $X, Celtics면 $Y 지급". 경기·마켓·리그별 집계 + 라이브 갱신.
- **왜:** 지금은 베팅을 *받기만* 하고 한 경기에 얼마 물렸는지 모른다 → 한 명/한 픽에 몰빵당해도 무방비.
- **데이터:** `positions`(status='open', stake, potential, meta.legs.gid/sel) — **새 수집 없이 집계만**.
- **산출:** 각 결과 시나리오별 `Σpotential(그 결과로 이기는 베팅) − Σstake(전체)`. 가장 아픈 결과(worst-case)를 빨갛게.

### 2. 리스크 기반 베팅 수락 (max liability + auto-refer)
- **무엇:** 마켓별 **최대 노출 한도**. 초과하면 배당 자동 인하 / 거절 / **검토 큐(refer)**. 큰 베팅·의심 베팅은 자동수락 말고 사람이 승인.
- **왜:** 한 명이 한도 없이 때릴 수 있는 지금 구조 = 사고의 씨앗.
- **데이터/위치:** `place_bet` RPC에 가드 1개 + 노출 뷰(1) 참조.

### 3. 손익(P&L) 대시보드 — handle / GGR / hold%
- **무엇:** **Handle**(총 베팅), **GGR**(=handle−지급), **Hold%**(=GGR/handle, 업계 5~8% 목표). 스포츠·리그·마켓·일별.
- **왜:** 북이 돈 버는지 한눈에. 없으면 계기판 없이 비행.
- **데이터:** `ledger`(bet 차감 / bet_won 지급) + `settlements`(pnl) — **이미 다 있음**, 집계만.

## 🟠 P1 — 출시 후 곧 (안 하면 샤프가 갉아먹음)

### 4. 플레이어 리스크 프로파일링 + CLV
- **무엇:** **CLV(Closing Line Value)** — 유저가 마감배당보다 좋은 가격에 꾸준히 잡으면 = **샤프**(장기적으로 우리가 짐) → 한도 조이기. 레크리에이션 vs 샤프 분류 → 한도·프로모 차등.
- **분석:** 유저별 hold%·turnover·win rate·**베팅한 배당 vs 마감 배당**.
- **데이터:** `positions`/`settlements` + 마감 시점 `sports_odds` 스냅샷(이건 새로 저장 필요할 수 있음).

### 5. 북 마진(vig) 실제 적용 검증
- **무엇:** "best book feed"를 그대로 쓰면 마진 0 → 장기 적자. 제공 배당에 **목표 마진(오버라운드)** 이 입혀지는지 검증.
- **분석:** 마켓별 **오버라운드 = Σ(1/decimal)**. 100% 초과분이 우리 마진. 목표(예 105~108%) 나오는지.

### 6. 이상 베팅 / 담합·매치픽싱 탐지
- **무엇:** 무명 경기 갑작스런 큰돈, 여러 계정 같은 픽(담합/칩덤핑), 신규계정 즉시 대액.
- **분석:** 경기별 베팅 급증 알림, 계정간 상관(같은 IP·기기·픽 타이밍).

## 📊 매일 봐야 할 분석 (대시보드 카탈로그)
1. **노출 히트맵** — 오늘/내일 경기별 worst-case 지급액 (P0-1)

6. **정산 정확도·지연** — 일일 감사 C1~C9 연계
7. **현금흐름** — 입출금·잔고 vs ledger (월간 명세서와 같은 소스)

## 🧱 데이터는 대부분 "이미 있다"
- 노출·P&L·hold%·CLV 기초는 **`positions`·`ledger`·`settlements`** 로 바로 — 일일 감사(`run_sports_audit`)와 **똑같은 패턴**(SECURITY DEFINER 집계 함수 + 백오피스 화면).
- 새로 저장이 필요한 것: **마감배당 스냅샷**(CLV용), **계정 메타**(IP/기기, 담합탐지용).

## ✅ 추천 착수 순서
1. **노출 뷰(P0-1)** — 데이터 있음, 임팩트 최대, 감사와 같은 패턴. 백오피스에 한 화면.
2. **P&L/hold% 대시보드(P0-3)** — "돈 버나" 매일.
3. **max liability 가드(P0-2)** — `place_bet`에 한 줄.
→ 이 셋이 끝나면 매일 메일에 *"감사 🟢 + 노출 $X + hold 6.2%"* 한 번에. **북을 손바닥 위에서.**
