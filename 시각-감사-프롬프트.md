# 시각/렌더 감사 프롬프트 — 정적이 못 잡는 "화면" 버그를 잡는다

> `verify.js`(정적: 파싱+node테스트+diagnose)는 **렌더를 안 한다.** 그래서 차트 왜곡·%
> 폭주·빈 화면·시뮬 흔들림을 **원리상 못 본다.** 이 프롬프트는 그 빈틈을 메운다 —
> **앱을 실제로 그려서 눈 + 단언(invariant)으로** 잡는다.
>
> 한 줄 정신: **"추론하지 말고 렌더해서 봐라."** 돈/차트는 특히. (RED→GREEN→**SHOW**의 SHOW)

---

## 0. 언제 도는가
- 차트·숫자·표·빈상태를 **건드린 모든 변경 후** (특히 crypto/FX 차트, 백오피스 목록, 명세서).
- 새 화면/위젯 추가 시. 릴리즈 전 1회 전수.

## 1. 원칙
1. **렌더 강제.** 코드만 읽고 "괜찮겠지" 금지. Playwright 헤드리스(이 환경에 설치됨) 또는 실기기로 **실제로 그린다.** 스크린샷을 남긴다.
2. **상태 매트릭스로 본다.** 한 화면을 여러 상태로: `(A)빈데이터 (B)포인트1~2개 (C)극단값(거대 입금/출금) (D)실시간 틱이 흐르는 중 (E)현금-only (F)변동성코인-heavy`. **버그는 거의 항상 극단/빈 상태에서 나온다.**
3. **가능하면 단언으로 박는다(자동).** 눈으로만 보는 체크는 다음 사람이 건너뛴다 → Playwright 단언으로 게이트에 물린다. 못 박는 건 체크리스트.

## 2. 실제로 친 시각 버그 클래스 (카탈로그 — 여기서부터 본다)
> 전부 **이번 세션에 실제로 발생**했다. 새 클래스를 칠 때마다 여기에 추가한다(§5).

| id | 증상 (실제) | 단언 / 체크 |
|---|---|---|
| **VIS-chart-amplify** | 변동 −0.00%(현금지갑)인데 라인이 **코너→코너 극적**으로. y축이 (min~max)를 꽉 채워 $35차이를 전체높이로 증폭 | 변동률 작으면(<0.5%) 라인 진폭이 화면높이의 **<30%**. (표시폭에 값의 ±1% 바닥) |
| **VIS-chart-jitter** | 차트 끝점이 **매 틱 위/아래로 뒤집힘**(시뮬 tip ±0.06%). "왔다갔다 지맘대로" | **입력 불변이면 2회 렌더 사이 SVG path 좌표 동일.** tip은 실데이터(anchorTotal), 시뮬값 금지 |
| **VIS-chart-collapse** | min==max(평평)에서 0나눗셈 → 라인이 **바닥/천장에 딱 붙음** | 평평해도 라인 y가 **[20%,80%]** 안(가운데). `(v-min)/(max-min)` 무방비 division 금지 |
| **VIS-pct-insane** | 변동 **−100.25%** / +$3.8M인데 부호·값 모순. 같은날 대형 출금이 baseline을 blow up | `changePct` **유한 & 합리범위**. baseline=series[0](실데이터). +값엔 ▲, −값엔 ▼ 일치 |
| **VIS-not-real-holdings** | **ETH를 대부분 보유**인데 차트가 ETH처럼 안 움직임(현금흐름 계단만). 반대로 현금인데 출렁 | 단기차트가 **내 실제 보유 × 그 코인 실시세**와 상관. 보유 없는 자산 모양 차용 금지 |
| **VIS-empty-silent** | 데이터 0인데 **빈 화면**(에러·로그인 안내 없음). 백오피스 players 안 뜸 | 결과 0/에러 시 **안내 텍스트 노출**("데이터 없음"/"어드민 로그인 필요"), 무음 빈칸 금지 |
| **VIS-fake-motion(돈)** 🚨 | 돈 화면에 **없는 움직임을 합성**(랜덤워크 / 시뮬틱 tip). #5 가짜데이터 | 차트 소스 = **실데이터**(snapshot/ledger/실시세). `Math.random`·시뮬tick을 돈차트 시리즈에 금지 |
| **VIS-number-jitter** | 큰 잔고 숫자가 시뮬 틱으로 **계속 까딱** | (정보) 라이브 시세는 OK지만, **안정 anchor**도 함께 보여 신뢰 깨지지 않게 |

## 3. 절차 (화면마다)
각 대상 화면(§6)에서:
1. **상태 매트릭스(A~F) 렌더.** 특히 차트는 (B)포인트1~2개, (C)거대 입금/출금, (E)현금-only, (F)코인-heavy 를 반드시.
2. 각 상태에서 **§2 카탈로그 단언**을 돌리고 **눈으로** 본다. 스크린샷.
3. 시계/실시간 위젯은 **두 번 렌더해 흔들림(jitter)** 확인 (입력 불변인데 좌표 변하면 VIS-chart-jitter).
4. **빈/에러 상태**를 일부러 만든다(권한 없음, 0행, 피드 차단) → 무음 빈칸이면 VIS-empty-silent.

## 4. 실행 도구 (Playwright — 자동화 팔)
- `tests/visual-smoke.js`(헤드리스): 각 페이지 로드 → **콘솔 에러/React 크래시 0** → 차트 SVG `path` 좌표 추출 → §2 단언(진폭/유한%/비-collapse/2회동일). 세션은 테스트계정 또는 모킹.
- 못 띄우는 화면이라도 **최소선:** 콘솔에러 0 + 표시 % 유한 + 빈상태 안내문 존재.
- 통과/실패를 `verify.js`가 호출하도록 물린다(설치된 Playwright/Chromium 사용, `PLAYWRIGHT_BROWSERS_PATH=/opt/pw-browsers`).

## 5. Control 루프 (식스시그마)
- 새 시각버그 → **이 §2 표에 한 줄**(클래스·증상·단언) + 가능하면 **Playwright 단언** 추가 → `verify.js`에 물림.
- "처음 보는" 클래스는 여전히 한 번은 눈으로 봐야 한다 — 그 1회를 **반드시 렌더로** 하고, 즉시 카탈로그에 박아 **재발은 자동**으로 막는다.

## 6. 대상 화면 카탈로그 (우선순위)
1. **crypto 지갑 차트** (1D~ALL × 빈/현금/ETH-heavy/틱중) ← 이번 버그 다발지
2. crypto 코인 행 **미니 스파크라인** (보유/미보유)
3. **명세서** `statement.html` (긴 내역/빈 달/거대 입금 계단)
4. **백오피스 목록** (players/online/accounts: 빈·로딩·권한없음)
5. FX 차트·포지션·마진 표시
6. sports 라이브 **시계/배당 티켓** (잠긴 배당 흔들림 #21)
7. 로그인/가입 폼 (에러바·OTP 빈칸)
