## 1. SQL 배포 (RPC·트리거·크론)
**위치:** Supabase 대시보드 → 좌측 **SQL Editor**.
1. 새 쿼리 탭 → 배포할 `.sql` 내용 붙여넣기.
2. **Run** (우하단). "Success" 뜨면 함수/트리거 생성됨.
3. ⚠️ "런 성공" = 컴파일 성공일 뿐. **작동은 §6에서 따로 검증.**
- 함수 교체는 `create or replace` 라 안전(같은 이름 덮어씀).
- 롤백 = 구버전 `create or replace` 다시 Run.

## 2. 엣지함수 배포 (sports-settle, stake-accrue, *-prices …)
**SQL editor로 안 됨.** 둘 중 하나:

**A) Supabase CLI (터미널)**
```
git pull                                  # repo 최신 코드 받기
supabase functions deploy <함수명>          # 예: sports-settle
```
- repo의 `supabase/functions/<함수명>/index.ts` 가 그대로 배포됨.
- 미리 1회: `supabase login` + `supabase link --project-ref grxnbgtfnaayeluenvqh`.

**B) 대시보드 에디터**
- 좌측 **Edge Functions** → 함수 선택 → 코드 전체 교체 → **Deploy**.
- 이 경우 Claude에게 "전체 코드 줘" 하면 붙여넣을 코드 제공.

## 3. 크론(pg_cron) 관리 — SQL editor에서
- 목록: `select jobname, schedule, active from cron.job order by jobname;`
- 교체: `select cron.schedule('<기존jobname>', '<스케줄>', $$ ... $$);` (같은 이름 = 덮어씀)
- 삭제: `select cron.unschedule('<jobname>');`
- ⚠️ **새 jobname으로 추가하면 중복 생김** — 항상 기존 이름 확인 먼저.
- 현재 잡: `sports-settle-5m`(*/5), `stake-accrue-daily`(0 0), 가격피드들(crypto/fx/stock/odds/games).

## 6. 검증 쿼리 모음 (배포 후 눈으로 확인)
```sql
-- 정산 살아있나 (재배포 후 3~5분)
select max(updated_at) as last_settle from settlements;

-- 크론 상태 (중복 없나 / active)
select jobname, schedule, active from cron.job order by jobname;

-- 베팅 저장값 (place_bet 서버재가격 + SGP haircut)
select local_id, potential,
       meta->'legs'->0->>'gid' g1, meta->'legs'->1->>'gid' g2,
       meta->'legs'->0->>'am' am1, meta->'legs'->1->>'am' am2
from positions where server='sports' order by updated_at desc limit 2;
-- → 같은경기 2다리면 potential이 단순곱의 75%여야(haircut). am은 서버 라인값.

-- 잔고 = 오프닝 + Σledger 검산
select acct_no, balance, bonus from accounts where server='sports';
select created_at, kind, amount, ref from ledger where acct_no='<ACCT>' order by created_at desc limit 25;
```

## 7. 롤백 절차
- **SQL 함수:** 구버전 `create or replace` 다시 Run. (구 `place_bet`는 `_deployed_rpcs.sql`에 보관.)
- **엣지함수:** 직전 코드로 다시 `deploy`. (git이면 `git checkout <이전커밋> -- supabase/functions/<fn> && deploy`.)
- **크론:** `cron.unschedule` 후 직전 스케줄 다시 등록.
- **증상별:**
  - 베팅이 "market/line not offered"로 다 막힘 → 구 `place_bet` 롤백 + Claude에 `live_games` 구조 전달.
  - 정산 503/멈춤 → CRON_SECRET·크론토큰 불일치. 토큰 다시 맞추거나 함수 fail-open 구버전 롤백.
