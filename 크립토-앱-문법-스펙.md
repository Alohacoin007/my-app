## 0. 아키텍처 한 눈에 (돈 진실의 소재)

- **크립토 돈은 `accounts.balance`/`ledger`가 아니라 `crypto_holdings(acct_no, asset, qty)`** — PK `(acct_no, asset)`, `asset='USDT'`이 USD 현금(1:1) (supabase/sql/crypto_trade.sql:7-14). 스테이킹은 `crypto_stakes`.
- 클라(`crypto-live.html`)의 `balances` state는 **USD 값(qty×price)** 을 담고 **서버에서 매 로드 adopt만** 한다 — localStorage에 돈 저장 금지 (crypto-live.html:13324-13328). `staked` state도 동일.
- 모든 잔고 이동 = **서버 RPC** (`crypto_trade`/`swap_crypto`/`stake_crypto`/`unstake_crypto`/`crypto_send_internal`/`app_transfer`) + 멱등 `ref`. 클라 폴백(client path)은 오프라인/무계정/서버가격없음일 때만.
- 서버 가격 = `prices(symbol, mid, updated_at)`, **신선도 게이트 120초** (crypto_trade.sql:77-79).
- **RPC가 유일한 쓰기 경로여야 하지만 `crypto_holdings`/`crypto_stakes`의 write-lock RLS는 주석 상태(미실행)** — §4.11 참조. 대시보드는 이 사실 위에서 설계해야 한다.

## 3. 포트폴리오·보유 문법

### 3.1 crypto_holdings 읽기(ADOPT) 계약
- **ADOPT 루프**(crypto-live.html:13808-13858): `from('crypto_holdings').select('asset,qty').eq('acct_no', acct)`(:13816). 각 행 value = `asset==='USDT' ? qty : qty × cryptoPriceOf(asset)`(:13831) — **매 idle 틱마다 재평가**(가격 추종). `window.__cbLocalTs` 기준 7초 busy 가드(:13815, 로컬 거래 중이면 adopt 중단). 8초 interval + Realtime `postgres_changes` 구독(:13851-13854).
- **WRITE-BACK은 폐쇄**(A4): `crypto_holdings`가 RLS admin/RPC 전용이라 클라 mirror는 죽은 코드(주석 crypto-live.html:13795-13799). 모든 변경은 RPC 경유. `cryptoQtySig`(:408)로 qty 시그니처 비교(가격드리프트 무시).
- **STAKES ADOPT**(crypto-live.html:13888-13935): `from('crypto_stakes').select('asset,usd,period,since,principal')`(:13900), 없으면 principal 제외 재시도(:13902). 멀티텀 키 `asset|period`(:13910-13911). Realtime 구독(:13929).

### 3.2 잔고 표시(서버 전용 원칙 구현부)
- `balances`/`staked`는 **React state(메모리)**, localStorage 저장 안 함(crypto-live.html:13329-13330,13324-13328). `serverBalances`(FX/Sports 풀)도 메모리(:13362-13366, `window.__srvBal` 미러).
- **히어로 총액**(Dashboard) = Σbalances + Σstaked.usd(크립토만) — `heroBase`(crypto-live.html:1682-1689). Account 총액 = 크립토 지갑 + FX(equity) + Sports 풀 — `baseTotal`(crypto-live.html:10191-10198).
- FX는 **현금이 아니라 equity** 표시: `effBal('fx',...)`(crypto-live.html:1522-1525)가 `alpexa.fxEquity`(localStorage, alpexa-sync.js:230-240가 pullBalances에서 기록)를 읽음.
- **`sync_crypto_balance` 트리거만 push**: 히어로 값 변동 시 3초 스로틀로 `rpc('sync_crypto_balance',{p_acct})`(crypto-live.html:1799) — 서버가 `crypto_holdings×prices(+stakes)`로 `accounts.balance(crypto)` 재계산(§4.9). 클라는 숫자 위조 불가(주석 :1797-1798).

### 3.3 포트폴리오 히스토리(실데이터 원칙 — 조작 금지)
- **Dashboard 차트**: `rpc('get_portfolio_history',{p_acct, p_range})`(crypto-live.html:1654), `r.data.ok && points`면 `pts.map(+p.v||0)`(:1656-1657). `histPts` null=로딩/[]=이력없음(:1645-1647).
- **priceSeries**(crypto-live.html:1810-1830): `1D`/`1W`만 실 sparkline×실보유+flat base, `anchorTotal` 앵커 — 그 외 range는 **합성 금지**, `[cur,cur]` 평평(:1836-1841).

## 6. 세션·신원 가드 (대시보드가 그대로 이식해야 할 게이트)

> 결함-로그 2026-07-15 두 건("유령 세션", "게스트 자동생성")이 이 섹션에서 나왔다. 대시보드가 **로그인/세션 게이트를 돈 가드와 같은 급으로** 복제해야 한다.

### 6.1 3중 로드 게이트 (crypto-live.html:149-185, `<head>` 인라인)
1. **`?reset=1`**: 로컬 크립토 캐시+alpexa.me 삭제 후 login으로(:149).
2. **`#alpexa=…` 신원 복원**: 홈스크린 설치가 Safari 계정 상속(iOS 스토리지 격리 대응)(:151).
3. **로그인 게이트(익명 거래 금지)**: `alpexa.me`에 `accts && (email||authId)` 없으면 `login.html`로(:154). ⚠️ **`alpexa.me`는 로컬 이름 태그일 뿐** — 이것만으로 authed 판정 금지(2026-07-15 유령세션).
4. **세션 실존 게이트**: `AlpexaSync.db.auth.getSession()`으로 실 세션 확인, 없으면 `login.html?expired=1`(:175-180). per-tab 플래그 `alpexa.sessChecked`로 루프 방지(:174,:178).
5. **신원 소유 게이트**: `AlpexaSync.assertIdentity()`(:183) — 세션 uid가 표시 계정을 소유하는지, 불일치 확정 시 `login.html?switch=1`(alpexa-sync.js:288-306). per-tab `alpexa.idChecked`.

### 6.2 세션 저장 위치 (무관용 — CLAUDE.md #5)
- 손님 세션 = **`sessionStorage`(탭별)**, `localStorage` 아님(alpexa-sync.js:24-29). 새 탭은 로그아웃 시작, 탭 내 새로고침은 유지. 어드민은 격리 `alpexa-admin-auth`(localStorage). **3곳(alpexa-sync.js·login.html·signup.html)이 반드시 같은 storage** — 하나만 달라도 로그인 루프.

### 6.3 게스트 자동생성 함정 (2026-07-15) — ⚠️ 대시보드 금지사항
- `AlpexaSync.me()`(alpexa-sync.js:41-51)는 `alpexa.me` 부재 시 **랜덤 게스트 신원(P-####, SP/CR/FX-######)을 반환**(단, **persist 안 함** :48). 판정 함수가 이 폴백을 신뢰하면 authed가 영원히 참이 됨(결함-로그 2026-07-15). **대시보드는 authed 판정에 `AlpexaSync.me()` 쓰지 말고 `localStorage.alpexa.me`를 직접 읽어라**(crypto-live.html이 실제로 그렇게 함 — :13235,:13247,:13746 등 직접 파싱).

### 6.4 자동 로그아웃·사인아웃
- **idle 자동 로그아웃**(alpexa-sync.js:315-384): 기본 15분+30초 경고, `auth.signOut()`+`alpexa.me` 등 제거→`login.html?switch=1&idle=1`. per-tab. 어드민 제외. 백그라운드 복귀 시 실경과시간 재확인(:374-382).
- **수동 사인아웃 `doSignOut`**(crypto-live.html:253-257): `auth.signOut()` + `alpexa.me`/`userName`/`userEmail` 제거 → `login.html?switch=1`. 잔고/이력은 안 지움(재로그인 복원). "Sign out all other devices" subPage 별도(§7 id='security-signout-all').
- **신원 자동 새로고침**(crypto-live.html:14605-14620): 로드마다 서버 `players`→`accounts`로 cust_id/acct 재작성(cust_id 드리프트 구조적 제거).

### 8.1 대시보드가 인지해야 할 "덜 만들면 안 되는" 정직한 사실
1. **지정가/스탑 체결 실행기 부재**(§3.4) — Open Orders는 표시/취소만. 대시보드가 지정가를 실체결처럼 노출하면 거짓 UX.
2. **Receive 시뮬입금 죽은 경로**(§6.8 = crypto-live.html:3458 onReceived 미호출, App onReceiveSimulated 14292 도달불가) — 입금은 고정 회사주소 표시 + 백오피스 파이프라인.
3. **AssetRow 틱 플래시 미구현**(§2.4) — 앱 복제만으론 가격 플래시 없음.
4. **미실행 RLS 잠금 3건**(§4.11) — crypto_holdings/crypto_stakes write-lock, 포트폴리오 스냅샷 크론이 주석. 대시폴리오 총액 차트가 flat일 수 있고, 클라 write 차단이 RLS가 아니라 "클라 코드가 안 함"에 의존.
5. **stake-accrue PATCH가 period 미포함**(§4.13) — 멀티텀 스테이크 이자 매칭 잠재 이슈.
6. **크립토 i18n·RG UI 부재**(§7.4) — 다국어/자기배제는 신규 작업(스포츠 앱과 다름).
7. **client path 폴백 분기**(§4.1) — 증권/ALPXS/오프라인은 서버가 못 매기면 클라 체결. 대시보드도 동일 분기 없으면 해당 자산 거래 불가.
