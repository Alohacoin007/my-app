# 마스터 감사 프롬프트 — 4앱 100% 완성도 (50년차 기준)

> 한 번 "돌리면" 4앱(Crypto·FX·Sports·BackOffice)을 5개 차원으로 전수 감사한다.
> 기존 도구를 오케스트레이션 + 보안/업계기능 카탈로그 추가. 결과 = 우선순위 발견목록.

## 0. 대상 & 차원
- **앱:** crypto-live.html · trading.html(FX) · sports-live.html · manager-mobile.html(백오피스)
- **차원 5:**
  1. **돈 정확성** (머니프린팅·이중차감·우회) → `tests/diagnose.js` + `tests/*.test.js`
  2. **데모/가짜 데이터** (진짜처럼 보이는 seed) → `tests/diagnose-ui.js` UI-demo-balance
  3. **죽은/미완성 기능** (보이는데 안 함) → `diagnose-ui.js` UI-stub + `UI-감사-프롬프트.md`(추적)
  4. **빠진 기능** (업계 표준 대비) → `기능-완성도-프롬프트.md` 카탈로그
  5. **보안** (키노출·인증·권한·입력검증) → 아래 보안 체크

## 1. 자동 스캔 (먼저 돌린다)
```
node tests/verify.js          # 파싱 + 돈 회귀테스트 + diagnose(돈) + diagnose-ui(데모/스텁)
node tests/diagnose-ui.js     # 데모/스텁 상세
```
보안 grep 스윕:
- `service_role|eyJ...JWT` (키 노출) · `eval(` · 하드코딩 주소(`bc1|0x|[13]…`)
- `if (CRON_SECRET &&` (fail-open 크론) · `.rpc(` 호출 vs RLS 잠금
- `innerHTML\s*=` (XSS) · `localStorage.setItem.*alpexa\.(balance|...)`(돈 로컬)

## 2. 추적 감사 (앱·화면마다 — 정적이 못 잡는 것)
각 앱의 모든 화면에서 `UI-감사-프롬프트.md` B1~B4 적용:
- 모든 **버튼/토글** → 핸들러 실동작? 죽은 것?
- 모든 **숫자** → 서버 실데이터? 가짜?
- 모든 **기능섹션** → 끝까지 연결? 스텁?
- **빈상태** → 가짜 기본값 누출?

## 3. 빠진 기능 (업계 표준 카탈로그 대조 — `기능-완성도-프롬프트.md`)
