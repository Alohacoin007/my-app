## 0. 한 줄 목표
크립토 앱(crypto-live.html)의 **기능 36종을 100% 이식**한 데스크톱 레전드 대시보드 — 스포츠 대시보드와 같은 토큰(`legend-ui.css --lgd-*`)·같은 위젯 엔진·같은 하네스 규율.

## 1. 돈 불변식 (무관용)
1. **표시 = 서버 읽기 전용.** crypto_holdings·crypto_stakes·prices·requests를 SELECT만. 돈을 localStorage에 저장하지 않는다 (CLAUDE.md #5).
2. **이동 = 기존 RPC만, 새 돈 경로 0개.** `crypto_trade`·`swap_crypto`·`stake_crypto`·`unstake_crypto`·`crypto_send_internal`·`app_transfer`·`sync_crypto_balance`·`verify_withdraw_otp` + 입출금은 `pushRequest`(백오피스 승인) 경유. 멱등 ref 접두 규약 그대로.
3. **앱의 사전 가드 전량 이식** (결함-로그 2026-07-15 "부분 복제" 클래스 방지): 잔고·한도·2FA 티어($1k/$5k)·허용목록 24h·컴플라이언스·이중탭 가드 — 스펙 §4의 화면별 체크리스트가 채점표.
4. **세션 가드 3중** 이식: alpexa.me 태그 + auth.getSession() 실존 + 게스트 자동생성 금지 (유령세션 2클래스 핀).
5. **가격 신선도 락스텝**: 서버 120초 stale 게이트와 동일 기준으로 거래 버튼 잠금(fail-closed). 지연 뱃지 15초.

## 3. 데이터 레이어 (데이터-스펙 §5 채택)
- `pricesStore`: Binance WS 직결(1차) + **Supabase Realtime prices 구독(신규 — 앱보다 ~8초 빠름)** + feed Edge 1초 캐시(폴백) + 8초 폴링(최후). 
- `holdingsStore` / `balancesStore` / `txnsStore`: 스포츠 gamesStore/moneyStore 패턴, 세션 가드 내장.
- 신선도 게이트: UI 뱃지 15s, 거래 잠금 120s (서버 crypto_trade와 동일 기준).

## 4. 구현 단계 (관문 = verify 🟢 + 단계별 E2E)
- **M1** 셸(위젯 엔진 이식)+데이터 레이어+W1~W4 (읽기 전용 — 돈 코드 0)
- **M2** W5/W6 거래 (crypto_trade/swap — READ-BEFORE-WRITE 표 승인 후)
- **M3** W10+S2/S3 돈 이동 (Send/Transfer/Funds/Withdraw + 2FA/OTP 관문)
- **M4** W7~W9 + 설정 패널
- **M5** 커버리지 감사(36행 diff=0) + `crypto-dashboard-fidelity.test.js`(핀/밴 18+) + 헤드리스 E2E 풀셋
- 각 M 완료마다 스크린샷 증거 + 게이트 로그 제시. 배포는 마지막에 한 번.

## 5. 기획 결정 사항 (스펙의 "정직한 사실" 7건 처리)
| 사실 | 결정 |
|---|---|
| 지정가/스탑 체결기 부재 | W8은 표시/취소 전용 + "Pending (manual)" 라벨 — 실체결처럼 보이게 금지. 체결기 신설은 별도 프로젝트 |

| 미실행 RLS 잠금 3건 | **사용자 배포 권고 SQL**로 기획에 포함 (crypto_holdings/crypto_stakes write-lock + 스냅샷 크론) — M1 전 배포 권장 |
| stake-accrue period 미매칭 | Edge 수정안 함께 제출 (멀티텀 이자 정확성) |
| i18n·RG 부재 | 범위 외 — 백로그 등재 |
| ALPXS client path | 대시보드 동일 분기 이식 (없으면 ALPXS 거래 불가) |
