# 결함 등록부 (Defect Register) — 버그 클래스 → 근본원인 → 재발방지

> Six Sigma 정신: **결함을 "측정"하고, 근본원인을 없애고, 재발을 "도구로" 막는다.**
> 새 버그를 칠 때마다 여기에 한 줄 추가하고, 가능하면 `tests/diagnose.js`에 체크를
> 넣는다 (정밀하게 — 오탐 내는 체크는 무시당해서 더 해롭다).
>
> **게이트:** `node tests/verify.js` = 인라인 파싱 + `tests/*.test.js` + `diagnose.js`.
> 🟢 아니면 "완료" 금지.

### #33 정산 고아 — settle이 "오늘 스코어보드"만 읽어 늦은 경기 영영 미정산 (2026-06-30)
- **증상:** 일일 감사 C1이 묵은 미정산 2건(축구 MLS, SP-726759) 잡음 — 경기는 끝났는데 36h+ open. **감사(마스터)가 버그를 잡아낸 첫 실사례.**
- **근본원인:** `sports-settle`의 ESPN 호출이 `/scoreboard`(파라미터 없음=**오늘 경기만**). 정산이 그 날 못 잡으면(크론 다운/늦은 종료) 다음날 경기가 피드에서 빠짐 → 결과 영영 못 받음 → 오픈 베팅 **고아**. 모든 종목 해당(축구·UFC 등 settle 적게 맞는 종목서 더 잘 보임).
- **수정:** scoreboard 호출에 **`?dates=<최근 6일>` catch-up 윈도우** → 늦게라도 정산(자가치유). 리그·UFC fetch 둘 다.
- **재발방지:** `tests/settle-catchup.test.js` — settle이 `scoreboard?dates=` 없이 "오늘만" 가져오면 **게이트 빨강**. ("버그는 마스터가 잡아야지" — 운영 C1 + 정적 게이트 둘 다.) 후속: 6일 넘게 지난 고아는 event-id 직접조회 or N일 후 자동 void(환불).

### #31 "Reset all demo data" footgun — 전체 localStorage 일괄삭제로 어드민 세션 날림 (2026-06-30)
- **증상:** 백오피스 Config의 "🗑 Reset all demo data" 버튼을 누르면 **백오피스가 로그아웃** → RLS가 데이터를 다 가려 **빈 화면**. ("백오피스에 아무것도 안 떠"의 진범으로 의심.) 메시지는 "Apps start fresh — Sports $50,000"라 **데이터를 지운 줄 오해**(공포).
- **근본원인:** `Object.keys(localStorage).forEach(...removeItem)`로 **`alpexa`로 시작하는 모든 키**를 쓸어담음 → 어드민 세션 키 `alpexa-admin-auth`(+세션토큰)까지 삭제. 돈이 localStorage에 있던 **옛 시절 잔재** 버튼(지금 돈은 서버전용이라 실데이터는 안 지워짐 — 즉 쓸모는 0, 위험만 남음).
- **수정:** 버튼·핸들러 **제거**. (서버 데이터는 애초에 안 건드렸으니 **실제 손실 0** — 어드민 재로그인이면 복구.)

## FX앱 플로팅 P&L 가짜 스프레드(항상 슬립페이지) (2026-07-05)
- **클래스: 화면 손익이 서버 실현손익과 다른 기준.** 플로팅 청산가를 클라가 `BUY→mid, SELL→mid+m.spread`(시뮬 스프레드, 전 클래스)로 계산. 그러나 서버 `fx_close.sql`은 **비FX(크립토/주식/지수)를 MID로 청산**(v_half는 cls='FX'만), FX는 mid∓half(spr_pts+markup_pts). → 크립토 매도가 열자마자 시뮬 스프레드만큼 **영구 가짜 손실**(실제 청산 땐 없음). 근본원인: 시뮬 스프레드(표시용)와 서버 스프레드(돈)라는 두 진실.
- **수정:** `trading.html` `fxClosePx(m,side)` = fx_close.sql v_close 정확 복제(실제 피드 spr_pts+markup_pts·pip 동일, 비FX는 half=0). 플로팅(3627)·로컬청산표시(3837) 두 곳이 같은 함수 사용(진실 한 곳). 증명: `tests/fx-floating-spread.test.js` (RED: 크립토 매도 가짜 −스프레드 → GREEN: client==server v_close, 크립토 mid 양방, FX mid∓half, JPY pip 일치, 신규 크립토 플로팅 0).
- **주의(락스텝):** pip/half 공식은 fx_close.sql·fx-prices Edge와 동일 유지. 서버가 fx_open을 bid/ask로 바꾸면(스프레드 온오픈) 이 헬퍼도 같이 갱신.
