# 결함 등록부 (Defect Register) — 버그 클래스 → 근본원인 → 재발방지

> Six Sigma 정신: **결함을 "측정"하고, 근본원인을 없애고, 재발을 "도구로" 막는다.**
> 새 버그를 칠 때마다 여기에 한 줄 추가하고, 가능하면 `tests/diagnose.js`에 체크를
> 넣는다 (정밀하게 — 오탐 내는 체크는 무시당해서 더 해롭다).
>
> **게이트:** `node tests/verify.js` = 인라인 파싱 + `tests/*.test.js` + `diagnose.js`.
> 🟢 아니면 "완료" 금지.

## 근본원인 공통분모 (진짜 적)
1. **"한 곳만 바뀌겠지" 가정** → 항상 **모든 읽기·쓰기 지점 grep** 먼저 (#2, #3, #4).
2. **추측을 증거로 착각** → 실행/쿼리로 **눈으로 확인** (#5, #6, 그리고 $40 "차이"가 실은 정상이었던 건).
3. **강제 안 되는 규칙** → 도구(게이트)로 강제 (#7).

## `tests/diagnose.js` — 결함 클래스 자동 스캔 (현재 6종)

| 체크 | 심각도 | 막는 것 |
|---|---|---|
| `A2-client-ledger-insert` | CRITICAL | 클라가 `ledger`에 insert |
| `A6-client-balance-update` | CRITICAL | 클라가 `accounts.balance` 직접 update (ledger 우회) |
| `SEC-service-role-key` | CRITICAL | service_role 키/JWT가 브라우저로 노출 |
| `ADDR-demo-crypto-address` | HIGH | 데모/하드코딩 코인주소 주입 |
| `EVAL-in-app` | HIGH | 배포 앱에 `eval(` |
| `B8-failopen-cron` | HIGH | 돈 엣지함수 fail-open 크론 시크릿 |

**검토됨 예외(허용):** manager-mobile 크립토 `accounts.balance` 직접쓰기 — 백오피스는 admin(RLS 허용), 크립토 진짜 돈은 `crypto_holdings`(commands 경로), `accounts.balance`는 표시 캐시.

### 새 버그를 칠 때 (Control 루프)
1. 위 표에 한 줄 추가 (클래스·근본원인).
2. **행위 버그** → `tests/*.test.js`로 RED→GREEN.
3. **패턴 버그**(정밀 regex로 표현 가능) → `diagnose.js`에 체크 추가.
4. `node tests/verify.js` 🟢 확인 후에만 "완료".
