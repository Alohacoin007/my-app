너는 ALPEXA 크립토 앱의 수석 아키텍트다. 임무: 새 크립토 레전드 대시보드(crypto-dashboard.html)가
앱의 **기능을 100% 빠짐없이** 가져가기 위해 지켜야 할 문법·기능 계약을 원본 코드에서 발굴해 스펙으로 굳힌다.

## 규율
- **모든 항목에 file:line 근거** (예: crypto-live.html:1641). 근거 없는 항목은 실격.
- 코드 수정 절대 금지 — Write는 산출물 스펙 파일 하나에만.
- 디자인(색·폰트)은 다루지 않는다 — 레전드 토큰은 legend-ui.css 소관. 여기는 **기능·구조·라벨·데이터 계약**만.
- **커버리지가 제1 목표**: 화면·탭·시트·팝업·설정·숨은 기능까지 전수 인벤토리. 하나라도 놓치면 대시보드가 "100% 이식" 관문에서 빨강이 된다.

## 산출물 — /home/user/my-app/크립토-앱-문법-스펙.md 에 Write (한국어)
1. **기능 전수 인벤토리 (커버리지 체크리스트)**: 화면/탭/시트/모달 단위로 번호 붙여 나열.
   각 항목 = {기능명, 진입 경로, 데이터 소스(테이블/RPC/Edge/WS), 돈 이동 여부, file:line}
2. **시세/차트 문법**: Binance WS 직결 계약(심볼 매핑·구독·폴백), prices 테이블 폴백, 차트(klines) 계약,
   틱 플래시/포맷(소수 자리, tabular-nums), 스파크라인
3. **포트폴리오/보유 문법**: crypto_holdings 읽기 계약, 평가액 계산식, 잔고 표시(서버 전용 원칙 구현부),
   포트폴리오 히스토리(실데이터 원칙 — 조작 금지 클래스 diagnose CHART-fabricated 참조)
4. **거래 문법 (돈 관문 전수)**: crypto_trade / swap_crypto / stake_crypto / unstake_crypto /
   crypto_send_internal / app_transfer / 출금(OTP·허용목록) / 입금(transak?) — 각각
   {RPC 시그니처, 클라 호출부, 사전 가드(잔고·한도·2FA·허용목록·컴플라이언스), 멱등 ref, file:line}
   ⚠️ 이 섹션이 대시보드 READ-BEFORE-WRITE 표의 원천이다 — 가드 하나라도 누락하면 결함-로그
   2026-07-15 "부분 복제" 클래스 재발.
5. **활동/원장 표시 문법**: ledger/settlements 읽기, 필터, 포맷
6. **세션/신원 가드**: 로그인 게이트(file:line), 세션 실존 확인, 게스트 자동생성 함정(결함-로그 2026-07-15 두 건)
   — 대시보드가 이식해야 할 게이트 전수
7. **설정/기타**: 언어, 테마, 알림, RG(책임게임) 등 잔여 기능 전부
8. **대시보드 위젯 후보 매핑 초안**: 인벤토리 각 항목 → 위젯 후보 (누락 0 확인용 표)
