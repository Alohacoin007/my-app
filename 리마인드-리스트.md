  - 테스트(trading-test.html)서 완전 검증 후 라이브 적용. 실시간이 너무 빨라서 더블클로즈할 틈도 없음 + 크레임 잠금이 최후 안전망.

- **앱 측 배선:** 오픈 티켓 풋터에서 `anyLive===false`(전 레그 시작 전)일 때만 "Cancel bet · refund $X" 버튼 표시 → cancel_bet RPC 호출(서버-우선: await, 성공해야 로컬 환불+제거, 실패시 알림). anyLive면 버튼 숨김(이미 계산값 있음).

**상태:** 사용자가 "일단 보류". 재개 시 A/B 방식부터 결정 → cancel_bet SQL 작성(사용자 배포) → 앱 배선.
