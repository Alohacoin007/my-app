## ✅ F. 이미 정상 / 안전 (확인됨 — 다시 안 건드림)
- `place_bet` · `cash_out` · `crypto_trade` · `swap_crypto` · `fx_open/close` = **서버권위 + 멱등 + auth.uid() 검증**
- `app_transfer` = 가장 견고한 머니 경로 (원자적, 멱등 ref)
- 서버 정산 `sports-settle` = 멱등 `betpay-id` + claim
- 웰컴보너스/리퍼럴 = 서버강제, 1인1회 PK
- **서비스키 유출 없음**, anon은 accounts/positions/settlements 못 읽음 (prices만 공개=정상)
- PWA/설치, 멀티디바이스 동기화, 라이브 스코어/배당, 베팅/팔레이/슬립, 입출금요청 흐름, 출금 무결성 가드(백오피스), 배당표시
- **`ledger.ref` UNIQUE 제약** (방금 추가 — 같은 ref 이중적용 DB 차단)

## 🔧 G. 닫는 순서 (임팩트 큰 것부터, 계약대로 RED→GREEN)
1. **A1 캐시백 익스플로잇 제거** — 유일하게 *지금 작동중인* 머니프린팅. 최우선.
2. **쓰기 잠금 SQL 1방** — `ledger`·`accounts`·`crypto_holdings`·`crypto_stakes` 쓰기를 admin/RPC 전용으로 → **A2·A4·A5·A6 동시 차단**.
3. **클라 정산/델타 경로 제거** — `syncSportsBal` 추측델타 삭제, 정산은 서버 전용 → A3·A8 차단. (각 경로 RED→GREEN)
4. **출금을 서버 RPC로** (`withdraw_hold` 멱등) → A7.
5. **파괴 페이지 배포제외 + 푸터 백오피스 링크 제거 + `crypto_trades` 읽기 제한 + 입금 더미주소 교체** → B1~B7.
6. **컴플라이언스** (KYC/나이/Geo/약관/책임도박 서버강제) → C1~C5. *실거래 트래픽 전 필수.*
7. **핵심 RPC 커밋 & 리뷰** (place_bet/app_transfer/트리거) → D6.
